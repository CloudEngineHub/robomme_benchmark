# 后台运行服务器说明

本目录提供了多种方式让服务器在关闭SSH连接后继续运行，并可以查看所有日志。

**注意**: 所有脚本都位于 `scripts/` 目录中，并自动使用 Micromamba 环境 `/data/hongzefu/maniskillenv1114`。

## 方法一：使用启动脚本（推荐）

这是最简单的方法，使用 `nohup` 在后台运行服务器，并自动激活 Micromamba 环境。

### 启动服务器

```bash
cd /data/hongzefu/historybench-v5.6/gradio
./scripts/start_server.sh
```

服务器会在后台运行，所有输出会保存到 `logs/server.log` 和 `logs/server_error.log`。

**脚本会自动：**
- 激活 Micromamba 环境 `/data/hongzefu/maniskillenv1114`
- 在后台运行服务器
- 保存所有日志到文件

### 查看实时日志

```bash
# 查看标准输出日志（实时）
tail -f logs/server.log

# 查看错误日志（实时）
tail -f logs/server_error.log

# 同时查看所有日志（实时）
tail -f logs/server.log logs/server_error.log

# 或者使用交互式脚本
./scripts/view_logs.sh
```

### 查看服务器状态

```bash
./scripts/status_server.sh
```

### 停止服务器

```bash
./scripts/stop_server.sh
```

## 方法二：使用 Screen（适合需要交互的场景）

Screen 允许你创建一个可以随时重新连接的会话。

### 安装 Screen（如果未安装）

```bash
# Ubuntu/Debian
sudo apt-get install screen

# CentOS/RHEL
sudo yum install screen
```

### 使用 Screen 运行

```bash
cd /data/hongzefu/historybench-v5.6/gradio

# 创建一个新的 screen 会话
screen -S historybench

# 激活 Micromamba 环境
source /data/hongzefu/maniskillenv1114/bin/activate

# 在 screen 会话中启动服务器
python main.py

# 按 Ctrl+A 然后按 D 来分离会话（detach）
# 服务器会继续运行
```

### 重新连接 Screen 会话

```bash
# 列出所有 screen 会话
screen -ls

# 重新连接到会话
screen -r historybench

# 如果只有一个会话，可以直接使用
screen -r
```

### 退出 Screen 会话

在 screen 会话中按 `Ctrl+A` 然后按 `K`，然后按 `Y` 确认。

## 方法三：使用 Tmux（Screen 的现代替代品）

Tmux 功能更强大，是 Screen 的现代替代品。

### 安装 Tmux（如果未安装）

```bash
# Ubuntu/Debian
sudo apt-get install tmux

# CentOS/RHEL
sudo yum install tmux
```

### 使用 Tmux 运行

```bash
cd /data/hongzefu/historybench-v5.6/gradio

# 创建一个新的 tmux 会话
tmux new -s historybench

# 激活 Micromamba 环境
source /data/hongzefu/maniskillenv1114/bin/activate

# 在 tmux 会话中启动服务器
python main.py

# 按 Ctrl+B 然后按 D 来分离会话（detach）
# 服务器会继续运行
```

### 重新连接 Tmux 会话

```bash
# 列出所有 tmux 会话
tmux ls

# 重新连接到会话
tmux attach -t historybench
```

### 退出 Tmux 会话

在 tmux 会话中输入 `exit` 或按 `Ctrl+B` 然后按 `X`。

## 日志文件位置

- **标准输出日志**: `logs/server.log`
- **错误日志**: `logs/server_error.log`

所有日志都会持续追加，不会覆盖之前的内容。

## 常用命令速查

```bash
# 启动服务器（后台运行，自动使用 Micromamba 环境）
./scripts/start_server.sh

# 停止服务器
./scripts/stop_server.sh

# 查看服务器状态
./scripts/status_server.sh

# 查看日志（交互式）
./scripts/view_logs.sh

# 查看实时日志（直接命令）
tail -f logs/server.log

# 查看最后100行日志
tail -n 100 logs/server.log

# 搜索日志中的关键词
grep "关键词" logs/server.log

# 查看日志文件大小
du -h logs/server.log
```

## 故障排查

### 如果服务器无法启动

1. 检查端口是否被占用：
```bash
lsof -i :7860
```

2. 查看错误日志：
```bash
cat logs/server_error.log
```

3. 检查 Python 环境：
```bash
python3 --version
which python3
```

### 如果无法停止服务器

1. 查看进程：
```bash
ps aux | grep "python3 main.py"
```

2. 手动停止：
```bash
kill <PID>
# 或强制停止
kill -9 <PID>
```

### 如果日志文件太大

可以清理旧日志（注意：这会删除所有日志）：

```bash
# 备份当前日志
mv logs/server.log logs/server.log.backup
mv logs/server_error.log logs/server_error.log.backup

# 创建新的空日志文件
touch logs/server.log logs/server_error.log
```

或者使用 logrotate 自动管理日志（需要配置）。

## 注意事项

1. **磁盘空间**: 日志文件会持续增长，请定期检查磁盘空间
2. **权限问题**: 确保脚本有执行权限（`chmod +x *.sh`）
3. **Python 路径**: 如果使用虚拟环境，需要在脚本中激活虚拟环境
4. **端口冲突**: 如果端口被占用，服务器会自动查找下一个可用端口
