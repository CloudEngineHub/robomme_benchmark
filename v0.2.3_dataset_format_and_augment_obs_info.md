# v0.2.3 数据集格式与 `_augment_obs_and_info` 输出说明

本文档包含两部分内容：

1. 数据集格式说明（沿用 `v0.2.3dataset_format.md` 的组织风格）。
2. `obs/info` 定义以 `robomme/env_record_wrapper/DemonstrationWrapper.py` 的 `_augment_obs_and_info` 为准。

---

## 1. 数据集格式

### 1.1 文件与目录结构

```text
<dataset_root>/
  hdf5_files/
    <EnvID>_ep<Episode>_seed<Seed>.h5   # 每个 episode 一个文件
  videos/                               # （可选）录制视频
```

- 录制由 `RobommeRecordWrapper` 完成。
- 仅成功 episode 会写入 HDF5。

### 1.2 HDF5 层级总览

```text
episode_<N>/
  timestep_<K>/
    obs/
    action/
    info/
  setup/
```

### 1.3 `obs/` 字段

| 字段 | 类型 / 形状 | 说明 |
|---|---|---|
| `front_camera_rgb` | `uint8 (H, W, 3)` | 正面相机 RGB |
| `wrist_camera_rgb` | `uint8 (H, W, 3)` | 腕部相机 RGB |
| `front_camera_depth` | `float (H, W, 1)` | 正面相机深度 |
| `wrist_camera_depth` | `float (H, W, 1)` | 腕部相机深度 |
| `joint_state` | `float64 (9,)` | 关节状态（7 关节 + 2 夹爪，不足补齐） |
| `eef_state` | `float64 (6,)` | 末端执行器 `[x, y, z, roll, pitch, yaw]` |
| `gripper_state` | `float64 (2,)` | 夹爪状态（`qpos` 后两维） |
| `gripper_close` | `bool` | 夹爪闭合标志 |
| `eef_velocity` | `float64 (6,)` | 末端线速度 + 角速度 |
| `front_camera_extrinsic` | `float (3, 4)` | 正面相机外参（OpenCV） |
| `wrist_camera_extrinsic` | `float (3, 4)` | 腕部相机外参（OpenCV） |

### 1.4 `action/` 字段

| 字段 | 类型 / 形状 | 说明 |
|---|---|---|
| `joint_action` | `float64 (8,)` 或 `"None"` | 关节动作（不足按规则补齐） |
| `eef_action_raw/pose` | `float64 (3,)` | 末端位置 |
| `eef_action_raw/quat` | `float64 (4,)` | 末端四元数 |
| `eef_action_raw/rpy` | `float64 (3,)` | 末端 RPY |
| `eef_action` | `float64 (7,)` | `[x, y, z, roll, pitch, yaw, gripper]` |
| `keypoint_action` | `float64 (7,)` | `[x, y, z, roll, pitch, yaw, gripper]` |

### 1.5 `info/` 字段

| 字段 | 类型 | 说明 |
|---|---|---|
| `record_timestep` | `int` | 当前帧序号 |
| `simple_subgoal` | `bytes (UTF-8)` | 子目标文本（planner 视角） |
| `simple_subgoal_online` | `bytes (UTF-8)` | 子目标文本（online 视角） |
| `grounded_subgoal` | `bytes (UTF-8)` | 带定位替换后的子目标文本 |
| `grounded_subgoal_online` | `bytes (UTF-8)` | online 版本定位子目标文本 |
| `is_video_demo` | `bool` | 是否演示阶段帧 |
| `is_keyframe` | `bool` | 是否 keyframe 刷新帧 |

### 1.6 `setup/` 字段

| 字段 | 类型 | 说明 |
|---|---|---|
| `seed` | `int` | 环境种子 |
| `difficulty` | `str` | 难度等级 |
| `task_goal` | `str` | 自然语言任务 |
| `front_camera_intrinsic` | `float (3, 3)` | 正面相机内参 |
| `wrist_camera_intrinsic` | `float (3, 3)` | 腕部相机内参 |
| `subgoal_list/subgoal_<i>_name` | `bytes` | 第 i 个子目标名称 |
| `subgoal_list/subgoal_<i>_demonstration` | `bool` | 第 i 个子目标是否演示阶段 |

### 1.7 实现要点

1. `eef_state` 与 keypoint 的姿态使用连续化 RPY 流水线。
2. `joint_state` 固定为 9 维，不足补 0。
3. `joint_action` 固定为 8 维，不足补 `-1`。
4. `keypoint_action` 按 keyframe 规则回填。
5. 相机内参写入 `setup/`，外参按时间步写入 `obs/`。

## 2. `_augment_obs_and_info` 输出协议

函数定义：

```python
_augment_obs_and_info(obs, info, action) -> (new_obs, new_info)
```

其中：

- `new_obs`：固定键集合的观测字典（9 个键）。
- `new_info`：在原始 `info` 基础上注入 8 个键。

### 2.1 `new_obs` 字段表（以函数实现为准）

| 键 | 来源表达式 | 语义 | 常见类型/形状 |
|---|---|---|---|
| `maniskill_obs` | `base_obs` | 原始 ManiSkill 观测 | `dict[str, Any]` |
| `front_camera` | `obs['sensor_data']['base_camera']['rgb'][0]` | 正面 RGB 帧 | `(H, W, 3)` |
| `wrist_camera` | `obs['sensor_data']['hand_camera']['rgb'][0]` | 腕部 RGB 帧 | `(H, W, 3)` |
| `joint_states` | `self.agent.robot.qpos` | 机器人关节状态 | 常见 `(9,)` 或等价 tensor |
| `velocity` | `(linear_v, angular_v)` | 末端线速度 + 角速度 | `tuple((3,), (3,))` |
| `front_camera_depth` | `obs['sensor_data']['base_camera']['depth'][0]` | 正面深度 | `(H, W, 1)` |
| `wrist_camera_depth` | `obs['sensor_data']['hand_camera']['depth'][0]` | 腕部深度 | `(H, W, 1)` |
| `end_effector_pose_raw` | `robot_endeffector_pose` | 末端原始姿态字典（含 pose/quat/rpy） | `dict[str, Tensor]` |
| `end_effector_pose` | `pose[:3] + rpy[:3]` | 末端 6 维姿态 | `list[6]`，即 `[x, y, z, roll, pitch, yaw]` |

说明（当前版本）：
- `front_camera_segmentation`、`front_camera_cam2world_opengl`、`wrist_camera_cam2world_opengl` 不再注入到 `new_obs`（源码中保留注释行）。
- `front_camera_extrinsic_opencv`、`front_camera_intrinsic_opencv`、`wrist_camera_extrinsic_opencv`、`wrist_camera_intrinsic_opencv` 已迁移到 `new_info`。

### 2.2 `new_info` 规则

`new_info` 由下面规则生成：

```python
new_info = {
    **info,
    'subgoal': subgoal_text,
    'subgoal_grounded': grounded_subgoal,
    'available_options': available_options,
    'language_goal': language_goal,
    'front_camera_extrinsic_opencv': base_camera_extrinsic_opencv,
    'front_camera_intrinsic_opencv': base_camera_intrinsic_opencv,
    'wrist_camera_extrinsic_opencv': wrist_camera_extrinsic_opencv,
    'wrist_camera_intrinsic_opencv': wrist_camera_intrinsic_opencv,
}
```

分为两类：

1. 透传字段（来自原始 `info`）
- 来自 `**info`，保持原键值。

2. augment 注入字段（固定 8 个）
- `subgoal`
- `subgoal_grounded`
- `available_options`
- `language_goal`
- `front_camera_extrinsic_opencv`
- `front_camera_intrinsic_opencv`
- `wrist_camera_extrinsic_opencv`
- `wrist_camera_intrinsic_opencv`

`available_options` 元素结构：

```python
{"action": str, "need_parameter": bool}
```

## 3. `evaluate.py` 在不同 `action_space` 下拿到什么

文件：`/data/hongzefu/robomme-v0.2b6/public_scripts/evaluate.py`

### 3.1 返回结构（四种 action_space 一致）

无论是 `joint_angle`、`ee_pose`、`keypoint` 还是 `oracle_planner`，`evaluate.py` 拿到的返回结构都一致：

- `reset()` 返回：`(obs_batch, info_batch)`
- `step()` 返回：`(obs_batch, reward_batch, terminated_batch, truncated_batch, info_batch)`

其中：

- `obs_batch` / `info_batch`：`dict[str, list]`（按时间步打包）
- `reward_batch`：`torch.float32` 一维张量
- `terminated_batch` / `truncated_batch`：`torch.bool` 一维张量

### 3.2 不同 action_space 的“输入动作”与“输出批量长度”

| action_space | `evaluate.py` 传入动作（用户视角） | `step()` 返回批量长度 |
|---|---|---|
| `joint_angle` | 关节动作向量（示例为 8 维） | 通常 `N=1` |
| `ee_pose` | 末端位姿动作 `[x, y, z, roll, pitch, yaw, gripper]` | 通常 `N=1` |
| `keypoint` | 关键点动作（至少前 7 维有效：`[x, y, z, roll, pitch, yaw, gripper]`） | 常见 `N>=1`（一次动作可产生多帧） |
| `oracle_planner` | 指令字典 `{"action": str, "point": [x, y]}` | 常见 `N>=1`（一次指令可产生多帧） |

### 3.3 `evaluate.py` 实际消费的 `obs/info`（来源仍以 `_augment_obs_and_info` 为准）

`obs_batch` 中可直接使用的核心键：

- `maniskill_obs`
- `front_camera`
- `wrist_camera`
- `front_camera_depth`
- `wrist_camera_depth`
- `end_effector_pose_raw`
- `end_effector_pose`
- `joint_states`
- `velocity`

`info_batch` 中可直接使用的核心键：

- `language_goal`
- `subgoal`
- `subgoal_grounded`
- `available_options`
- `front_camera_extrinsic_opencv`
- `front_camera_intrinsic_opencv`
- `wrist_camera_extrinsic_opencv`
- `wrist_camera_intrinsic_opencv`
- 以及透传的状态键（如 `success`、`fail`）

`evaluate.py` 中通常通过 `v[-1]` 取当前批次最后一帧进行判断与打印。

---

## 4. 公共接口与类型约定

1. `_augment_obs_and_info(obs, info, action) -> (new_obs, new_info)`
2. `new_obs: dict[str, Any]`，键集合固定为文档列出的 9 个键
3. `new_info: dict[str, Any]`，结构为“透传字段 + 8 个 augment 注入字段”

---

## 5. 验收清单

1. 文档中 `new_obs` 9 个键与 `_augment_obs_and_info` 完全一致。
2. 文档中 `new_info` 注入键与 `_augment_obs_and_info` 完全一致。
3. 读者可仅依赖本文档解析 augment 输出，不需要从 `evaluate.py` 反推字段定义。
